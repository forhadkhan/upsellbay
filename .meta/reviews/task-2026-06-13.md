Review both the original PRD (`.meta/PRDs/UpsellBay-PRD-v1.md`) and the latest PRD (`.meta/PRDs/UpsellBay-PRD-v4.md`).

First, identify how the current implementation differs from both documents. Note that the current implementation has already progressed beyond many areas covered in the final PRD, so do not limit your analysis to the documented requirements. Evaluate the actual product state independently.

The primary concern is that UpsellBay is still not delivering meaningful value to merchants or shoppers:

* The admin panel is partially functional and relatively mature, although it still requires UX, workflow, and feature improvements.
* The storefront/customer-facing experience is currently unclear, fragmented, and provides little practical value. This is the highest-priority area for analysis.

Your task is to perform a deep product review and produce a comprehensive gap analysis focused on making UpsellBay genuinely useful and competitive.

### Competitor Research

Study leading WooCommerce upsell, cross-sell, product recommendation, post-purchase, bundle, and offer plugins, including but not limited to:

* FunnelKit
* CartFlows
* WooCommerce Product Recommendations
* Frequently Bought Together plugins
* WPFunnels
* YayCommerce Upsell plugins
* ShopBrain
* ReConvert
* Bold Upsell
* Zipify
* One Click Upsell
* Any other relevant market leaders

For each competitor:

* Identify what makes it valuable to merchants.
* Identify what makes it valuable to shoppers.
* Analyze how offers are surfaced throughout the customer journey.
* Identify patterns that are considered industry-standard.
* Highlight opportunities where UpsellBay can differentiate itself.

Note that, we want to keep UpsellBay lean, to the point, solves one problem at best that all other are not solving or not in the best way. Maybe old PRDs mentioned this.

### Storefront Experience Analysis

Treat the storefront as the most important part of the review.

Determine:

* What customer-facing experiences are missing.
* Which offer placements are actually useful versus intrusive.
* Which offer types generate measurable value.
* How recommendations should behave across:

  * Product page
  * Cart page
  * Mini cart
  * Checkout
  * Thank-you page
  * Post-purchase flow
  * Order details page
  * Customer account area

Evaluate:

* Offer visibility
* Relevance
* Timing
* UX friction
* Mobile experience
* Performance impact
* Accessibility
* Conversion impact

For every recommendation, explain:

* Why it matters.
* How competitors handle it.
* How UpsellBay should implement it.

### Offer Management & Conflict Resolution

Design a complete offer governance model.

#### Multiple UpsellBay Offers on the Same Product

Define how the system should behave when:

* A product already has an active offer.
* A product has multiple active offers.
* A product has paused offers.
* A product has draft offers.
* A merchant creates a new offer targeting a product that already participates in other offers.

Specify:

* Validation rules.
* Conflict detection rules.
* Priority system.
* Scheduling rules.
* Eligibility evaluation order.
* Merchant warnings.
* Auto-resolution options.
* Manual override options.

Determine whether:

* Multiple offers can run simultaneously.
* Only one offer should be shown.
* Offer stacking should be allowed.
* Priority-based selection should be used.

### External Offer Conflicts

Analyze conflicts with:

* Native WooCommerce sales and discounts.
* WooCommerce coupons.
* WooCommerce product bundles.
* WooCommerce composite products.
* WooCommerce subscriptions.
* WooCommerce dynamic pricing plugins.
* Discount plugins.
* Funnel plugins.
* Other upsell/cross-sell plugins.

For each scenario:

* Define expected behavior.
* Define conflict detection logic.
* Define merchant-facing warnings.
* Define customer-facing behavior.
* Prevent double-discounting and offer duplication.

### WooCommerce Architecture Review

Ensure every recommendation follows WooCommerce best practices.

Review:

* WooCommerce CRUD APIs.
* Product APIs.
* Cart APIs.
* Checkout APIs.
* Session handling.
* Order APIs.
* HPOS compatibility.
* Block-based checkout compatibility.
* Store API compatibility.
* WooCommerce extension guidelines.

Avoid solutions that bypass WooCommerce architecture when official APIs and extension points already exist.

For every major feature, specify:

* Recommended WooCommerce hooks.
* Recommended WooCommerce APIs.
* Data storage strategy.
* Compatibility considerations.
* Performance considerations.
